VaultAPI

Secure Document Vault REST API
v1.4.2 | JWT-Authenticated | HS256
API Endpoints
GET /api/docs Public
List all available API endpoints with descriptions and parameters.
POST /api/register Public
Register a new user account and receive a JWT access token.
{ "username": "string", "password": "string" }
POST /api/login Public
Authenticate with credentials and receive a JWT access token.
{ "username": "string", "password": "string" }
GET /api/profile Auth Required
View your user profile and JWT claims. Requires valid Bearer token.
GET /api/vault Auth Required
List documents in the vault. Admin role reveals additional classified documents.
GET /admin Admin Only
Admin dashboard. Requires JWT with role: "admin" claim.
POST /admin/greeting Admin Only
Set a custom greeting message template for the admin panel.
template=Hello {{ username }}, welcome back!
Quick Start

1. Register an account:

curl -X POST http://localhost:8080/api/register \
  -H "Content-Type: application/json" \
  -d '{"username": "myuser", "password": "mypass"}'

2. Copy the token from the response.

3. Access authenticated endpoints:

curl http://localhost:8080/api/profile \
  -H "Authorization: Bearer <your-token>"

4. Explore the vault:

curl http://localhost:8080/api/vault \
  -H "Authorization: Bearer <your-token>"
Authentication

VaultAPI uses JSON Web Tokens (JWT) with the HS256 algorithm for authentication. After registering or logging in, include the token in the Authorization header as a Bearer token.

Tokens expire after 24 hours. The JWT payload includes the user and role claims for authorization.
